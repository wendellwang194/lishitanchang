# 离石弹唱视听体验

一个以离石弹唱为核心的互动网页作品。  
观众点击空白处，唱词会伴随声音“生长”出现；播放结束后，文本进入带物理堆积感的场域，形成一幅可听、可读、可触的动态版式。

在线体验：<https://wendellwang194.github.io/lishitanchang/>

## 产品亮点

- 视听联动：点击触发唱词与音频同步出现
- 诗性呈现：逐字出现、浮动过渡、整体氛围偏大地色文艺风格
- 物理堆积：文本可碰撞、可旋转、可拖拽再播放
- 单声道优先级控制：同一时刻仅播放一个音频，避免嘈杂叠音
- 清场与静音：支持一键清场、静音/解除静音
- 自动资源更新：监听 `audio/` 目录，自动生成最新音频清单

## 快速开始

### 1. 准备音频

将音频文件放入 `audio/` 文件夹。  
页面文本将自动使用“文件名（去后缀）”作为唱词内容。

### 2. 生成音频清单

手动执行一次：

```bash
node build-manifest.js
```

### 3. 打开页面

直接打开 `index.html` 即可体验。  
如果你在开发调试，建议使用本地静态服务器并配合自动监听（见下文）。

## 音频清单更新（推荐自动模式）

运行：

```bash
node watch-manifest.js
```

保持该终端窗口开启后，`audio/` 内新增、删除、重命名文件时，`audio/manifest.js` 会自动更新。  
更新后刷新浏览器即可看到新内容。

## 交互说明

- 点击空白处：触发新一条唱词与对应音频
- 拖拽已有文本：重播该文本对应音频，松手后继续物理掉落
- 右上角静音按钮：静音/解除静音
- 页面上方清场按钮：随机渐隐清除当前文本堆积

## 项目结构

- `index.html`：页面结构
- `style.css`：视觉样式与动画
- `script.js`：交互、音频调度、物理系统
- `audio/`：音频资源目录
- `audio/manifest.js`：音频清单（自动生成）
- `build-manifest.js`：手动生成清单
- `watch-manifest.js`：监听目录并自动更新清单
- `font/`：项目使用字体资源

## 部署

本项目已支持 GitHub Pages，默认使用 `main` 分支根目录发布。  
如需重新发布，推送代码到 `main` 即可触发页面更新。

## 常见问题

- 新增了音频但页面没反应：
  1. 先执行 `node build-manifest.js`（或保持 `node watch-manifest.js` 运行）
  2. 强制刷新页面：`Cmd + Shift + R`
- 音频可见但无法播放：
  1. 检查浏览器是否阻止自动播放
  2. 先进行一次用户点击交互再测试
